При виконанні завдання звертати увагу
- на ті повідомлення, що виводить GIT у консоль.
  Потрібно проаналізувати кожне повідомлення
- на зміст опису комміту!

1. У каталозі проектів створити каталог Home_1 та
скопіювати в нього файл .gitingnore з каталогу 01_html
  
2. У створеному каталозі створити файл index.html і додати до нього 1 заголовок h1 та 2 параграфи тексту (використовувати генератор lorem ipsum...)

3. У створеному каталозі створити репозиторій git,
додати файл index.html у буфер, а потім зробити commit цього файлу в репозиторій
4. До файлу index.html додати 3-й параграф та перевірити статус репозиторію

5. Закоментувати файл index.html у репозиторій та перевірити статус репозиторію

6. Повторити ще раз пункти 4, 5

7. У каталозі Home_1 створити файл catalog.html
    Перевірити статус репозиторію - проаналізувати повідомлення git

8. Додати файл catalog.html до stage area (буфер) і знову перевірити статус репозиторію.
Порівняти повідомлення з попереднім та проаналізувати у чому різниця

9. Подивитись історію коммітів
   Якщо Ви все зробили правильно, то у Вас виведуться 3 логи коммітів
